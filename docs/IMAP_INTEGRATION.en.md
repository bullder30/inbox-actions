# IMAP Integration

This documentation describes the IMAP integration for email connection in Inbox Actions (Gmail, Yahoo, iCloud, Fastmail, ProtonMail...).

---

## Overview

For non-Microsoft providers, Inbox Actions uses **IMAP with App Password** to access emails. This method is universal and works with all email providers.

| Provider | Recommended method |
|----------|--------------------|
| **Microsoft (Outlook, Hotmail, Live)** | Microsoft Graph API ([see docs](./MICROSOFT_GRAPH.en.md)) |
| **Gmail, Yahoo, iCloud, Fastmail, ProtonMail...** | IMAP with App Password |

### Mutual Exclusivity

**Important**: Only one email provider can be active at a time.

- **Connecting IMAP** → Automatically removes the Mail.Read scope from Microsoft Graph
- **Connecting Microsoft Graph** → Automatically removes IMAP credentials

This exclusivity avoids conflicts and ensures only one provider is used for synchronization.

## Architecture

### Factory Pattern

The system uses a Factory pattern to abstract the email provider:

```
┌─────────────────────────────────────┐
│  createEmailProvider(userId)        │  ← Single entry point
│  lib/email-provider/factory.ts      │
└──────────────┬──────────────────────┘
               │
      ┌────────┴────────┐
      ▼                 ▼
┌──────────────┐  ┌──────────────┐
│ MicrosoftGraph│  │ IMAPProvider │
│   Provider    │  │              │
└──────────────┘  └──────────────┘
```

The provider implements `IEmailProvider`:

```typescript

  countNewEmails(): Promise<number>;
  disconnect(): Promise<void>;
  getStatus(): Promise<ConnectionStatus>;

```

## IMAP Configuration

### Supported Providers

| Provider | IMAP Server | Port | TLS |
|----------|-------------|------|-----|
| Gmail | imap.gmail.com | 993 | Yes |
| Yahoo | imap.mail.yahoo.com | 993 | Yes |
| iCloud | imap.mail.me.com | 993 | Yes |
| Fastmail | imap.fastmail.com | 993 | Yes |
| ProtonMail | 127.0.0.1 (via Bridge) | 1143 | No |

> **Note**: For Microsoft (Outlook, Hotmail, Live), use [Microsoft Graph API](./MICROSOFT_GRAPH.en.md) — no IMAP configuration needed.

### Automatic Detection

The system automatically detects the provider from the email address:

```typescript
// lib/imap/types.ts
export function detectProviderFromEmail(email: string): string | null {
  const domain = email.split("@")[1]?.toLowerCase();

  if (domain === "gmail.com") return "gmail";
  if (domain === "yahoo.com") return "yahoo";
  if (domain === "icloud.com" || domain === "me.com") return "icloud";
  // Microsoft → redirect to Graph API
  if (["outlook.com", "hotmail.com", "live.com"].includes(domain)) {
    return "microsoft"; // Signal to use Graph API
  }
  return null;
}
```

## Security

### Password Encryption

IMAP passwords are encrypted with **AES-256-CBC**:

```typescript
// lib/imap/imap-credentials.ts
export function encryptPassword(plainPassword: string): string {
  const masterKey = getMasterKey(); // IMAP_MASTER_KEY env var
  const iv = crypto.randomBytes(16);
  const cipher = crypto.createCipheriv("aes-256-cbc", masterKey, iv);
  // ...
}
```

### Required Environment Variable

```bash
# Generate a master key (64 hex characters = 32 bytes)
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Add to .env.local
IMAP_MASTER_KEY=<your_64_character_key>
```

### App Password Setup

An **App Password** is a special password generated by your email provider. It allows IMAP access without using your main password.

#### Gmail

1. Enable 2-step verification on your Google account
2. Go to [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
3. Select "Mail" and "Other (custom name)"
4. Name it "Inbox Actions"
5. Copy the generated password (16 characters, no spaces)
6. **Important**: Enable IMAP in Gmail → Settings → See all settings → Forwarding and POP/IMAP

#### Yahoo Mail

1. Enable 2-step verification
2. Go to [login.yahoo.com/account/security](https://login.yahoo.com/account/security)
3. "Generate app password"
4. Select "Other app"
5. Copy the generated password

#### iCloud Mail

1. Enable two-factor authentication
2. Go to [appleid.apple.com](https://appleid.apple.com)
3. "Security" → "App-Specific Passwords"
4. Click "+" to generate a new password
5. Copy the generated password

#### Fastmail

1. Go to Settings → Passwords & Security
2. "New app password"
3. Create a password for "IMAP"
4. Copy the generated password

#### ProtonMail

ProtonMail requires **ProtonMail Bridge** for IMAP access:

1. Download and install [ProtonMail Bridge](https://protonmail.com/bridge)
2. Sign in via Bridge
3. Click your account to see IMAP credentials
4. Use the password shown in Bridge
5. **Host**: 127.0.0.1, **Port**: 1143, **TLS**: No

## Data Model

### IMAPCredential Table

```prisma
model IMAPCredential {
  id              String   @id @default(cuid())
  userId          String

  // IMAP configuration
  imapHost        String   // e.g. "imap.gmail.com"
  imapPort        Int      @default(993)
  imapUsername    String   // Email or IMAP username
  imapPassword    String   @db.Text // AES-256 encrypted

  // Configuration
  imapFolder      String   @default("INBOX")
  useTLS          Boolean  @default(true)
  authMethod      IMAPAuthMethod @default(PASSWORD)

  // Sync tracking
  lastIMAPSync    DateTime?
  lastUID         BigInt?  // Last synced UID

  // Connection status
  isConnected     Boolean  @default(false)
  connectionError String?
  lastErrorAt     DateTime?

  user User @relation(...)

  @@unique([userId, imapHost, imapUsername])
}
```

### EmailMetadata

```prisma
model EmailMetadata {
  // Provider
  emailProvider  EmailProvider

  // IMAP identifiers
  imapUID        BigInt?
  imapMessageId  String?

  // Common metadata
  from           String
  subject        String?
  snippet        String
  receivedAt     DateTime
  labels         String[]

  @@unique([userId, imapUID])
}
```

## Troubleshooting

### "Authentication failed" Error

1. Verify the App Password is correct (no spaces)
2. For Gmail: verify that IMAP access is enabled in Gmail settings
3. Verify that 2-step verification is enabled on your account
4. Some providers block connections from new applications — check your inbox for a security alert

### "Connection refused" Error

1. Verify the host and port
2. Verify that TLS is enabled (port 993)
3. Check your firewall / antivirus
4. For ProtonMail: verify that Bridge is running

### "IMAP_MASTER_KEY not set" Error

Generate and configure the encryption key:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
# Add the result to .env.local
IMAP_MASTER_KEY=<your_key>
```

### Gmail: "IMAP access is not enabled"

1. Go to Gmail → Settings (gear icon)
2. "See all settings"
3. "Forwarding and POP/IMAP" tab
4. Enable "Enable IMAP"
5. Save

### Microsoft Email Detected

If you enter an @outlook.com, @hotmail.com or @live.com address, the IMAP form will redirect you to Microsoft Graph API which provides a better user experience.
